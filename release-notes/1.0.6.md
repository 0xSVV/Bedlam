Routing, DNS, and reconnect fixes.

- IPv6 set to Disabled now blocks v6 traffic instead of letting it bypass the VPN. Bypass-only remains the mode that routes v6 outside the tunnel.
- System DNS sends your network's public resolvers through the tunnel and falls back to Cloudflare when there are none. Before, it either leaked queries to the local network or broke name resolution.
- The tunnel notices Wi-Fi and mobile switches while the VPN is up and re-dials right away instead of waiting out a timeout.
- Routing and per-app filter changes apply in place; the connection no longer drops while they take effect.
- Reconnects re-resolve the server address, and a tunnel that sends but receives nothing is probed and re-dialed within a minute.
- If reconnecting fails for three minutes, the VPN stops and posts a notification that traffic is no longer protected.
- Tunnel DNS servers stay inside the tunnel even when a direct-route source covers their addresses.
- Imported links with an invalid port are rejected instead of silently connecting on 443. The lazy switch, which did nothing, is gone.
